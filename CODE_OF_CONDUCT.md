# Code of Conduct

FailPack community spaces should be useful, respectful and safe for developers.

This applies to GitHub issues, discussions, documentation feedback, support channels and other public FailPack spaces.

## Expected behavior

Please:

- be respectful,
- stay constructive,
- describe problems clearly,
- keep feedback technical and useful,
- avoid spam,
- avoid harassment,
- report security issues privately.

## Unacceptable behavior

Do not use FailPack spaces for:

- harassment or abuse,
- hate speech,
- spam or scams,
- personal attacks,
- doxxing,
- sharing private data,
- posting secrets or stolen credentials,
- intentionally disruptive behavior.

## Enforcement

Maintainers may edit, hide or remove content that breaks these rules.

Maintainers may also lock conversations, close issues or block users who abuse FailPack spaces.

## Security reports

Do not post sensitive security details publicly.

Use:

```text
security@failpack.dev
```
